Lensa - UX Engineer

By integrating front-end developers (UX engineers, as we called ourselves) into the design team we improved the efficiency of designer-developer communication and made sure that the developed components were the same as the designers imagined them in our design system.

Project Goals

  • Separate design system team only focusing on components.
  • Develop a reusable, framework-agnostic design system for developers.

My Tasks

  • Collaborate with the designers and other development teams.
  • Create web components based on the design system.
  • Hand over the finished components to the development teams.
  • Updating the current, React and Svelte based component library.

Highlighted Project

Web Component Library

We initiated the creation of a web component-based library for our developer teams. While web components have been around for some time, their integration with certain frameworks (e.g., React) still poses challenges. To maximize compatibility, we selected Lit as our web components framework.

Our close collaboration with the designers allowed us to quickly iterate on the developed web components, accelerating our design decisions. This approach enabled the developer teams to focus on building new features, reducing the need to revisit design issues and inconsistencies.

Used technologies:

  • HTML, CSS, Tailwind
  • Lit Web Components
  • React
  • Svelte
  • Storybook

Retro

What went well?

  • The concept and workflow of the design system, along with the roles of UX engineers, significantly increased the speed and quality of our component development.

What didn't go well?

  • At times, unforeseen technical issues arose, halting our development progress.

What have I learned?

The role of the UX Engineer, which combines design sensibility with development skills, greatly enhanced the efficiency and quality of our component development.
